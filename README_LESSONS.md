# درسنامه شیءگرایی در پایتون (نسخه کامل و مرتب)

این یک راهنمای خودمونی و روان برای درک مفاهیم اصلی برنامه‌نویسی شیءگرا (OOP) در پایتون است. هدف اینه که با مثال‌های ساده، این مفاهیم رو یک بار برای همیشه یاد بگیری.

---

### ۱. کلاس (Class) و شیء (Object)

**کلاس (Class):**
مثل یک **نقشه ساخت** یا **قالب شیرینی‌پزی** است. کلاس، طرح کلی یک مفهوم را تعریف می‌کند اما خودش یک چیز واقعی و قابل استفاده نیست. در واقع، کلاس مشخص می‌کند که یک "چیز" چه ویژگی‌ها (Attributes) و چه رفتارهایی (Methods) باید داشته باشد.

**شیء (Object):**
یک **نمونه واقعی** است که از روی کلاس ساخته می‌شود. مثل ساختمانی که از روی نقشه ساخته شده یا شیرینی که با قالب درست شده. در پایتون، تقریباً همه‌چیز یک شیء است (اعداد، رشته‌ها، لیست‌ها و...).

```python
# مثال: تعریف یک کلاس ساده برای ماشین
class Car:
    # این یک کلاس خالی است که فعلاً هیچ ویژگی یا رفتاری ندارد
    pass

# ساخت دو شیء (نمونه) از روی کلاس Car
car1 = Car()
car2 = Car()

print(car1)  # <__main__.Car object at 0x...>
print(car2)  # <__main__.Car object at 0x...> (در یک آدرس حافظه دیگر)
```

### ۲. ویژگی‌های نمونه (Instance Attributes) و تابع سازنده (`__init__`)

**ویژگی (Attribute):**
اطلاعات یا داده‌های مربوط به یک شیء خاص هستند. مثلاً برای یک ماشین، ویژگی‌ها می‌توانند مدل، رنگ و سرعت باشند.

**روش اشتباه (و دردسرساز):**
می‌توان ویژگی‌ها را به صورت دستی بعد از ساخت شیء به آن اضافه کرد. این کار توصیه نمی‌شود چون کد را شلخته می‌کند و احتمال خطا (AttributeError) را بالا می‌برد.

```python
car1.model = "Pride"  # این کار اصولی نیست!
print(car1.model)

# اگر فراموش کنیم ویژگی را تعریف کنیم، برنامه کرش می‌کند:
# print(car2.model)  # AttributeError: 'Car' object has no attribute 'model'
```

**راه درست (استفاده از `__init__`):**
تابع `__init__` که به آن **سازنده (Constructor)** هم می‌گویند، یک متد جادویی است که به محض ساخته شدن یک شیء جدید، به طور خودکار فراخوانی می‌شود. بهترین جا برای تعریف ویژگی‌های اصلی یک شیء، همین تابع است.

```python
class Car:
    def __init__(self, model, color):
        print(f"یک ماشین مدل {model} با رنگ {color} ساخته شد!")
        self.model = model
        self.color = color

# حالا هنگام ساخت شیء، باید اطلاعات اولیه را به آن بدهیم
car1 = Car("Tesla Model S", "Red")
car2 = Car("BMW X5", "Black")

print(f"مدل ماشین اول: {car1.model}")  # خروجی: Tesla Model S
print(f"رنگ ماشین دوم: {car2.color}")   # خروجی: Black
```

### ۳. متدهای نمونه (Instance Methods) و پارامتر `self`

**متد (Method):**
توابعی هستند که **داخل کلاس** تعریف می‌شوند و رفتارها یا عملیات مربوط به اشیاء آن کلاس را مشخص می‌کنند. مثلاً یک ماشین می‌تواند متد "گاز دادن" یا "ترمز کردن" داشته باشد.

**پارامتر `self`:**
`self` نماینده **خودِ آن شیء** است که متد را فراخوانی کرده. وقتی شما می‌نویسید `car1.accelerate()`، پایتون به صورت خودکار خودِ `car1` را به عنوان آرگومان اول به متد می‌فرستد. به همین دلیل `self` همیشه اولین پارامتر در تعریف متدهای نمونه است. این پارامتر به متد اجازه می‌دهد به ویژگی‌ها و سایر متدهای همان شیء دسترسی داشته باشد.

```python
class Car:
    def __init__(self, model, speed=0):
        self.model = model
        self.speed = speed

    # این یک متد نمونه است
    def accelerate(self, amount):
        self.speed += amount
        print(f"ماشین {self.model} سرعتش به {self.speed} کیلومتر بر ساعت رسید.")

    # این هم یک متد دیگر
    def brake(self, amount):
        self.speed -= amount
        if self.speed < 0:
            self.speed = 0
        print(f"ماشین {self.model} ترمز کرد. سرعت فعلی: {self.speed} کیلومتر بر ساعت.")

car1 = Car("Ferrari")
car1.accelerate(50)  # خروجی: ماشین Ferrari سرعتش به 50 کیلومتر بر ساعت رسید.
car1.accelerate(30)  # خروجی: ماشین Ferrari سرعتش به 80 کیلومتر بر ساعت رسید.
car1.brake(20)       # خروجی: ماشین Ferrari ترمز کرد. سرعت فعلی: 60 کیلومتر بر ساعت.
```

### ۴. ویژگی‌های کلاس (Class Attributes)

این ویژگی‌ها مستقیماً **داخل بدنه کلاس** (خارج از همه‌ی متدها) تعریف می‌شوند و بین **تمام اشیاء ساخته‌شده** از آن کلاس **مشترک** هستند. از آن‌ها برای نگهداری داده‌هایی استفاده می‌شود که به یک نمونه خاص تعلق ندارند، بلکه به کل کلاس مربوطند.

**مثال:** شمارش تعداد کل ماشین‌های ساخته شده.

```python
class Car:
    # این یک ویژگی کلاس است
    number_of_cars = 0

    def __init__(self, model):
        self.model = model
        # هر بار که یک شیء جدید ساخته می‌شود، این ویژگی کلاس را یکی زیاد می‌کنیم
        Car.number_of_cars += 1

print(f"تعداد کل ماشین‌ها قبل از ساخت: {Car.number_of_cars}")  # خروجی: 0

car1 = Car("Pride")
car2 = Car("Peykan")

print(f"تعداد کل ماشین‌ها بعد از ساخت: {Car.number_of_cars}")   # خروجی: 2
print(f"تعداد ماشین‌های car1: {car1.number_of_cars}")          # خروجی: 2 (از طریق شیء هم قابل دسترسی است)
```

**نکته مهم (Shadowing):**
اگر مقدار یک ویژگی کلاس را از طریق **خود کلاس** تغییر دهید (`Car.number_of_cars = 10`)، این تغییر برای تمام اشیاء اعمال می‌شود. اما اگر آن را از طریق یک **شیء** تغییر دهید (`car1.number_of_cars = 5`)، پایتون یک **ویژگی نمونه جدید** فقط برای همان شیء `car1` ایجاد می‌کند و ویژگی کلاس اصلی دست‌نخورده باقی می‌ماند.

### ۵. متدهای ایستا (Static Methods)

این‌ها توابعی هستند که داخل کلاس تعریف می‌شوند اما **هیچ ارتباطی با شیء (`self`) یا کلاس (`cls`) ندارند**. در واقع مثل یک تابع معمولی هستند که برای نظم و ترتیب بیشتر، داخل کلاس قرار گرفته‌اند. بالای این متدها از دکوراتور `@staticmethod` استفاده می‌شود.

**چه زمانی استفاده می‌شود؟**
وقتی یک تابع کمکی دارید که از نظر منطقی به کلاس مربوط است، اما برای کار کردن نیازی به اطلاعات شیء یا کلاس ندارد.

```python
class MathHelper:
    @staticmethod
    def add(x, y):
        return x + y

    @staticmethod
    def is_even(number):
        return number % 2 == 0

# برای فراخوانی نیازی به ساخت شیء نیست
result = MathHelper.add(5, 3)
print(result)  # خروجی: 8

is_num_even = MathHelper.is_even(10)
print(is_num_even)  # خروجی: True
```

### ۶. چهار اصل معروف شیءگرایی (The 4 Pillars of OOP)

**۱. انتزاع (Abstraction):**
- **یعنی:** پنهان کردن پیچیدگی‌های داخلی و ارائه یک رابط کاربری ساده.
- **مثال:** شما برای استفاده از تلویزیون، فقط از دکمه‌های ریموت کنترل استفاده می‌کنید و کاری به مدارهای الکترونیکی و نحوه پردازش سیگنال‌ها ندارید. در کدنویسی هم، وقتی از یک متد استفاده می‌کنید، فقط ورودی را می‌دهید و خروجی را می‌گیرید، بدون اینکه درگیر جزئیات پیاده‌سازی آن شوید.

**۲. کپسوله‌سازی (Encapsulation):**
- **یعنی:** قرار دادن داده‌ها (ویژگی‌ها) و کدهایی که روی آن داده‌ها کار می‌کنند (متدها) در یک واحد به نام "کلاس" و محدود کردن دسترسی مستقیم به داده‌های حساس.
- **مثال:** در یک کلاس `BankAccount`، شما اجازه نمی‌دهید هر کسی بتواند موجودی حساب (`balance`) را مستقیماً تغییر دهد. به جای آن، متدهایی مثل `deposit()` و `withdraw()` ارائه می‌دهید که قوانین خاصی (مثل عدم برداشت بیش از موجودی) را اعمال می‌کنند.

**۳. ارث‌بری (Inheritance):**
- **یعنی:** یک کلاس جدید (فرزند) می‌تواند تمام ویژگی‌ها و متدهای یک کلاس دیگر (والد) را به ارث ببرد و علاوه بر آن، ویژگی‌ها و متدهای جدیدی برای خودش تعریف کند یا موارد به ارث برده را بازنویسی (Override) کند. این کار از تکرار کد جلوگیری می‌کند.
- **مثال:**
  ```python
  class Animal:
      def speak(self):
          print("Some generic sound")

  class Dog(Animal):  # Dog از Animal ارث می‌برد
      def speak(self):  # بازنویسی متد speak
          print("Woof!")

  class Cat(Animal):  # Cat هم از Animal ارث می‌برد
      def speak(self):
          print("Meow!")

  dog = Dog()
  dog.speak()  # خروجی: Woof!
  ```

**۴. چندریختی (Polymorphism):**
- **یعنی:** "یک نام، چندین شکل". این اصل به اشیاء مختلف اجازه می‌دهد تا به یک پیام یا فراخوانی یکسان، به روش‌های متفاوت و متناسب با خودشان پاسخ دهند.
- **مثال:** در مثال بالا، هم `Dog` و هم `Cat` متد `speak` دارند. شما می‌توانید یک لیست از حیوانات مختلف داشته باشید و متد `speak` را برای همه آن‌ها فراخوانی کنید، و هر کدام صدای مخصوص خود را تولید خواهند کرد.
  ```python
  animals = [Dog(), Cat()]
  for animal in animals:
      animal.speak()  # یک بار Woof! و یک بار Meow! چاپ می‌شود
  ```
این اصل باعث می‌شود کد شما انعطاف‌پذیرتر و توسعه‌پذیرتر باشد.
